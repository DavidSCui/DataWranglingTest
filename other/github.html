GitHub Concepts

Repository

Repositories are like a project‚Äôs profile page - it‚Äôs where all the files related to a project is stored, as well as a track of file history.

Branches

You may notice a button saying ‚Äúmain‚Äù on top left of the files when looking at a repository. Clicking on this will show different branches of the project - these are like different workspaces within the project. Cloning

Nerdy way of saying ‚Äúdownloading literally the whole project and all the files‚Äù.

Pull

Pulling means downloading just the latest updates of the project, keeping your local files up to date with what‚Äôs been happening in the project without downloading the whole thing.

Push

Pushing means uploading your version of the project‚Äôs files (or just a single file) to GitHub.

Commits

Commits are like making savefiles or checkpoints - when making a push, you can make a note of what you‚Äôve changed so everyone can track what the change was. It also means that if something goes wrong with your new code, you can go back to an earlier commit (an earlier checkpoint) to backtrack.

Git#

Favourite insult, but also the tool you will need to work with GitHub - download Git here.

Logging into Git

Configure Git (set up your name and email): Before using Git, you need to identify yourself so that your contributions are properly recorded. Run these commands in the terminal:

sh  git config --global user.name "Your Name"  git config --global user.email "your.email@example.com"

Replace ‚ÄúYour Name‚Äù with your actual name and ‚Äúyour.email@example.com‚Äù with your email address.

Authenticate with GitHub: GitHub no longer allows direct password authentication for security reasons. Instead, you need to use a Personal Access Token (PAT).
Go to GitHub, navigate to Settings > Developer settings > Personal Access Tokens, and generate a new token.
When Git asks for your password, use the token instead.

Note

You can also do this for the most part in VSCode, so if the above looked like effort, look below. At some point, VSCode will automatically ask you to login.

Cloning the Repository#

  1. Pick a convenient workspace/directory to code in (e.g., a new folder in your Documents folder).

  2. Click the big search bar at the top of VSCode, and click ‚ÄúShow and Run Commands‚Äù.

If someone else changes stuff in the repository, you can update your version of the files.

Traditional GitHub Workflow:

The traditional way to do all of these steps are to do it in the terminal. This might end up being more intuitive for you. The core workflow is.

  1. Cloning: git clone + repository URL

  2. Staging: git add + either the filename OR . to stage everything.

  3. Commit: git commit -m + a brief description of what you‚Äôve done

  4. Pushing: git push (if you need to push to a specific branch, git push origin + branch)

  5. Pulling: git pull (if you need to pull from a specific branch, git pull origin + branch)
